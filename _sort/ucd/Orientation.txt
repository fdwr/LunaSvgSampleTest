# Property: Orientation

#  All code points not explicitly listed for Orientation
#  have the value Unknown (Unknown).

# @missing: 0000..10FFFF; Unknown

# ================================================
# This file supplies per-codepoint orientation information for supporting
# various text layout directionalities, particularly to resolve the
# occurrences of horizontal within vertical text.
#
# GDI and GDI+ only have two classes for orientation:
#
# 1. Characters that remain upright.
# 2. Characters that follow the baseline.
#
# For supporting stacked text or the other proposed UAX #9 approaches,
# two classes is insufficient. Though, for the simpler clients who only
# wants the two possible rotations, all the classes below can be mapped
# to just upright or rotated.
#
# In addition to rotation, the class data contains enough information
# to distinguish mirroring characters and scripts too. Most mirrorable
# characters (notably common punctuation) already have a substitution
# available via character translation; but a handful of scripts also
# mirror based on direction, and these are unlikely to ever receive
# (nor should they) mirror codepoints. Instead, these may be mirrored
# via a graphic transform or font feature, such as rtlm. The data does
# not mandate the approach that should be used, just that it is valid
# for the given script character to behave so.
#
# Attributes:
# D  - discrete (generally freely rotatable and stackable)
# C  - cursive/connected (not stackable, as it would look broken)
# S  - stackable
# R  - rotatable
# RF - rotatable, typically via font (the 'vert' or 'vrt2' feature)
# M  - mirrorable (via graphic transform or font feature 'ltrm'/'rtlm')
# MP - mirrorable via character substitution mirror pair (such as { } < >}
# I  - inherits orientation from previous (combining marks and variation selectors)
# 
# Classes:
# DS    - discrete stackable (ideographic upright, stacked, never rotated or mirrored, like Chinese)
# DSR   - rotatable and stackable discrete (Thai/Hebrew/most punctuation)
# DSRM  - rotable/stackable/historically mirrorable and in certain context (Greek/Latin)
# DRU   - discrete rotating/can rotate upside-down 180 (Rongo Rongo)
# DSM   - discrete stackable and mirrorable (Egyptian hieroglyphs/Mayan/Runic/(rarely)Orkhon)
# DRMP  - rotated punctuation/symbol (rotated in stacked TTB, such as "(", not via a vert feature)
# DRFMP - full width rotating punctuation with mirror pairs (typically stay upright and rotate via a 'vert' feature)
# DRF   - full width rotating punctuation (typically stay upright and rotate via a 'vert' feature)
# CR    - connected rotating (Devanagari/Arabic/Ogham)
# CRU   - connected rotating/can fully rotate even upside-down 180 (Mongolian/Rongo Rongo)
# CSR   - connected rotatable/stackable (Devanagari can actually be stacked on signs, though very rare)
# ADS   - standard arrows (maintain orientation regardless of text direction, unlike half-width)
# ADSR  - half-width arrows that rotate U+FFE9
# BOX   - Box drawing shapes (special class)
# BLK   - Block elements (special class)
# ANY   - neutral orientation
# CI    - connected to previous character, inheriting orientation
#
# The unknown codepoints are advised to recieve treatment like CR, to play it
# safe and not break new cursive scripts. Though, DSR will likely be fine too
# for most new scripts being added (as many older scripts are discrete).
